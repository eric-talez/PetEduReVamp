# 📦 Replit 프로젝트에 바로 붙여넣기

아래를 그대로 프로젝트 루트에 복붙하세요.

### 1) `vite.config.ts`

```ts
import { defineConfig } from 'vite';

export default defineConfig(({ mode }) => {
  const plugins: any[] = [];

  // ✅ Replit 전용 플러그인: 개발 모드 + 플래그 켤 때만
  if (mode === 'development' && process.env.VITE_ENABLE_REPLIT_PLUGINS === '1') {
    // eslint-disable-next-line @typescript-eslint/no-var-requires
    const modal = require('@replit/vite-plugin-runtime-error-modal');
    plugins.push(modal());
  }

  return {
    plugins,
  };
});
```

---

### 2) 클라이언트 엔트리 (`src/main.ts` 또는 `src/index.ts`)

```ts
// ✅ Replit 전용 에러 모달: 개발 + 플래그 켤 때만 로드
if (import.meta.env.DEV && import.meta.env.VITE_ENABLE_REPLIT_PLUGINS === '1') {
  import('@replit/vite-plugin-runtime-error-modal')
    .then(m => m.default?.())
    .catch(() => {
      // 프로덕션에서는 모듈이 없어도 그냥 무시
    });
}
```

---

### 3) `package.json` (scripts 부분 교체)

```json
{
  "scripts": {
    "build": "vite build && esbuild server/index.ts --platform=node --bundle --format=esm --outfile=dist/index.js --external:@replit/*",
    "start": "NODE_ENV=production node dist/index.js",
    "prestart": "npm run build",
    "preflight": "node scripts/preflight.js",
    "ext:diag": "node scripts/ext-diag.js",
    "smoke": "node scripts/smoke.js",
    "release:check": "npm run preflight && (npm run ext:diag || echo '(no ext)') && npm run build && npm run smoke",
    "migrate": "prisma migrate deploy || drizzle-kit migrate || knex migrate:latest || echo \"no migrator\""
  },
  "devDependencies": {
    "@replit/vite-plugin-runtime-error-modal": "^1.0.0"
  }
}
```

### 4) `scripts/preflight.js` (없으면 새로 생성)

```js
// scripts/preflight.js
console.log("🔍 Running preflight checks...");

// 1) prod에서 Replit 플러그인 플래그 켜짐 방지
if (process.env.NODE_ENV === 'production' && process.env.VITE_ENABLE_REPLIT_PLUGINS === '1') {
  console.error("[preflight] ❌ Do not enable Replit plugins in production");
  process.exit(1);
}

// 2) 로컬 DB 차단
if (process.env.DATABASE_URL && /127\.0\.0\.1|localhost/.test(process.env.DATABASE_URL)) {
  console.error("[preflight] ❌ Localhost DB URL is not allowed in production");
  process.exit(1);
}

// 3) DB_ENGINE vs URL 스킴 일치 확인
if (process.env.DB_ENGINE && process.env.DATABASE_URL) {
  if (process.env.DB_ENGINE === 'postgres' && !process.env.DATABASE_URL.startsWith('postgres')) {
    console.error("[preflight] ❌ DB_ENGINE mismatch: expected postgres");
    process.exit(1);
  }
  if (process.env.DB_ENGINE === 'mysql' && !process.env.DATABASE_URL.startsWith('mysql')) {
    console.error("[preflight] ❌ DB_ENGINE mismatch: expected mysql");
    process.exit(1);
  }
}

console.log("✅ Preflight checks passed");
```

### 5) Replit Secrets (환경변수)

* 개발용:

  ```
  NODE_ENV=development
  VITE_ENABLE_REPLIT_PLUGINS=1
  PORT=3000
  DATABASE_URL=postgres://dev-url
  DB_ENGINE=postgres
  DB_SSL=0
  SESSION_SECRET=devsecret123
  ```
* Railway/AWS에는 **VITE\_ENABLE\_REPLIT\_PLUGINS는 넣지 마세요** (혹은 0으로 지정)

---

### 6) 사용법

1. Replit 개발:

   ```
   npm run start
   ```

   👉 개발 모드라 Replit 플러그인 동작

2. Railway/AWS 빌드 확인:

   ```
   NODE_ENV=production VITE_ENABLE_REPLIT_PLUGINS=0 npm run build
   node dist/index.js
   ```

   👉 `@replit/*` 참조 없이 실행됨

3. 번들 검사:

   ```
   grep -R "@replit" dist || echo "OK: no replit refs"
   ```

---

## 🚀 효과

* Replit에서는 **에러 모달 그대로 사용**
* Railway/AWS 프로덕션 빌드에서는 **전혀 포함되지 않음**
* `preflight`로 실수(플래그 켬) 방지
* CI/CD(`release:check`)에서 빌드/헬스/네트워크 검사 후 배포 진행
