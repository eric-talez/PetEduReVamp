좋습니다. 그럼 바로 **테일즈용 강아지 코 인증 카메라 기능**을
**Replit + Spring Boot + MediaPipe(또는 Python 비전 서비스) + 모바일 카메라** 기준으로
실제로 개발 가능한 수준으로 정리해드리겠습니다.

핵심은 하나입니다.

> **QR은 방문 세션 확인**
> **코 인증은 “진짜 그 강아지인지” 확인**

즉, **2단계 인증 구조**로 가는 것입니다.

---

# 1) 전체 구조

테일즈에서 가장 안전하고 구현 가능한 방식은 **Spring Boot 메인 서버 + Python 비전 마이크로서비스** 분리형입니다.

## 권장 아키텍처

```text
[모바일 웹/태블릿 카메라]
        ↓
[React / Next 프론트]
        ↓
[Spring Boot API]
        ↓
 ┌───────────────────────┐
 │  Visit Session 서버   │
 │  QR 토큰 검증         │
 │  회원/반려견/예약 관리 │
 └───────────────────────┘
        ↓
[Python Vision Service]
  - 얼굴/코 ROI 추출
  - 이미지 품질검사
  - nose embedding 생성
  - 유사도 비교
        ↓
[DB / Vector 저장소]
```

# 3) 핵심 사용자 흐름

## A. 최초 등록

보호자가 반려견을 등록할 때:

1. 반려견 기본정보 입력
2. 반려견 등록번호 입력
3. 접종 정보 입력 또는 연동
4. 코 사진 5~10장 촬영
5. 시스템이 가장 좋은 프레임 선별
6. nose embedding 생성
7. DB 저장

---

## B. 매장 방문

1. 예약 생성
2. visit_session_token 발급
3. QR 생성
4. 입장 시 QR 스캔
5. 카메라로 강아지 코 스캔
6. 등록된 nose embedding과 비교
7. 일치하면 입장 승인

---

## C. 예외 처리

* QR은 맞지만 코 인증 실패 → 직원 수동 확인
* 코 인증은 맞지만 접종 만료 → 입장 제한
* 동일 QR 재사용 → 무효
* 다른 강아지가 대신 입장 → 차단

# 4) 데이터 모델 설계

아래 구조는 바로 Spring Boot 엔티티 설계로 옮길 수 있습니다.

## Pet

```java
@Entity
@Table(name = "pets")
public class Pet {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long ownerId;
    private String name;
    private String breed;
    private LocalDate birthDate;
    private String registrationNo;
    private String gender;
    private Double weightKg;

    private LocalDate rabiesValidUntil;
    private LocalDate vaccineValidUntil;

    private Boolean active;
}
```

---

## PetNoseProfile

```java
@Entity
@Table(name = "pet_nose_profiles")
public class PetNoseProfile {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long petId;

    @Lob
    private String embeddingJson;   // 벡터를 JSON 문자열로 저장하는 초기 버전

    private String representativeImageUrl;
    private Double qualityScore;
    private Integer version;

    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

---

## VisitSession

```java
@Entity
@Table(name = "visit_sessions")
public class VisitSession {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long petId;
    private Long ownerId;

    @Column(unique = true, nullable = false)
    private String qrToken;

    private LocalDateTime reservedAt;
    private LocalDateTime expiresAt;
    private Boolean singleUse;
    private Boolean used;
    private LocalDateTime usedAt;

    private Boolean qrVerified;
    private Boolean noseVerified;

    private String zonePermissions; // JSON or CSV
    private String status; // PENDING, APPROVED, DENIED, EXPIRED
}
```

---

## NoseVerificationLog

```java
@Entity
@Table(name = "nose_verification_logs")
public class NoseVerificationLog {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long visitSessionId;
    private Long petId;

    private Double similarityScore;
    private Boolean matched;
    private String capturedImageUrl;
    private String failReason;

    private LocalDateTime verifiedAt;
}
```

```http
GET /api/visit-sessions/{sessionId}/status
```

# 7) MediaPipe의 실제 역할

여기서 중요한 점이 있습니다.

MediaPipe를 바로 “강아지 신원 확인 완성품”처럼 쓰기보다,
**영상에서 좋은 프레임과 코 주변 ROI를 추출하는 전처리 엔진**으로 쓰는 편이 현실적입니다.

즉 역할은 아래처럼 잡으시면 됩니다.

## MediaPipe 담당

* 얼굴 대략 위치 찾기
* 주둥이/코 중심 프레임 선별
* 흔들림 적은 프레임 선택
* 이미지 크롭 보정

## 커스텀 모델 담당

* 코 패턴 특징 추출
* 임베딩 생성
* 등록된 개체와 비교

---

# 8) MVP에서는 어떤 모델이 현실적인가

처음부터 대형 AI 모델을 만들 필요는 없습니다.

## 1단계 MVP

* 코 영역 crop
* grayscale/contrast normalize
* CNN 임베딩 모델
* cosine similarity 비교

## 2단계 고도화

* Siamese Network
* ArcFace 계열 metric learning
* 위조 사진 탐지
* 다각도 보정

즉 처음에는:

> **“좋은 프레임 선별 + 코 패턴 벡터화 + 유사도 비교”**

이 정도만 되어도 충분히 MVP가 나옵니다.

# 10) 유사도 판정 기준 예시

초기에는 아주 단순하게 갈 수 있습니다.

```text
similarity >= 0.85  -> 승인
0.75 ~ 0.84         -> 재촬영 권장
< 0.75              -> 불일치
```

이 기준은 나중에 실데이터 쌓이면 조정하면 됩니다.

```java
    public NoseVerifyResult verify(Long petId, MultipartFile image, String storedEmbeddingJson) {
        // multipart/form-data로 Python Vision Service 호출

        return new NoseVerifyResult(true, 0.89, "OK");
```
