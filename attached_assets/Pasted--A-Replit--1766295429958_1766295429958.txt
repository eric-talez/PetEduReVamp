# 1️⃣ 테일즈 수익 구조 – 기본 원칙 (운영 안정화 중심)

### 핵심 원칙 3가지

1. **고정비 선회수**

   * 임대료 / 인건비 / 관리비 / 시스템비 먼저 확보
2. **훈련사 최소 보장**

   * 수익화 초기에도 “콘텐츠 생산 동기” 유지
3. **성과 기반 가변 지분**

   * 유튜브처럼 **성장하면 자동으로 비율이 바뀜**
   * 협상/재계약 최소화

# 3️⃣ 매출 단계별 지분 구조 (자동 전환)

## 📊 매출 구간별 분배 비율 (예시)

### 🔴 Stage 1 : 안정화 단계 (월 매출 < 500만원)

> *목표: 버티기*

| 항목       | 비율      |
| -------- | ------- |
| 플랫폼(테일즈) | **60%** |
| 훈련사      | **40%** |

* 고정비 회수 최우선
* 훈련사 최소 보장 (콘텐츠 생산 유지)

---

### 🟠 Stage 2 : 성장 단계 (월 매출 500~1500만원)

| 항목  | 비율      |
| --- | ------- |
| 플랫폼 | **50%** |
| 훈련사 | **50%** |

* 이 구간부터 **훈련사 “받는 느낌”이 확실**
* 이탈 방지 핵심 구간

---

### 🟢 Stage 3 : 확장 단계 (월 매출 1500만원 이상)

| 항목  | 비율      |
| --- | ------- |
| 플랫폼 | **40%** |
| 훈련사 | **60%** |

* 유튜브 상위 크리에이터 구조
* 장기 파트너십 유도

> ⚠️ 단, **운영 리스크 발생 시 Stage 하향 가능 (자동 규칙)**

# 4️⃣ 훈련사별 개인 분배 공식 (공정성 핵심)

```text
훈련사 지급액 = (훈련사 풀) × (개인 기여도 점수 / 전체 훈련사 점수 합)
```

### 기여도 점수 = 기존 코드의 `trainerScore`

이미 구현됨 ✅
→ **조회 / 시청 / 좋아요 / 채택 답변 / 팔로워 증가**

---

### 정산 알고리즘 (의사코드)

```js
function settle(monthRevenue) {
  const stage = getStage(monthRevenue.total_amount);

  const platformRatio = stage.platform;
  const trainerRatio = stage.trainer;

  const trainerPool = monthRevenue.total_amount * trainerRatio;

  const trainers = getTrainersWithScore();
  const totalScore = sum(trainers.score);

  trainers.forEach(t => {
    const payout = trainerPool * (t.score / totalScore);
    savePayout(t.id, payout);
  });
}
```

> 이 로직은 **사람 개입 없이 월 1회 자동 실행 가능**

# 6️⃣ 운영 안정화를 위한 안전장치 (중요)

### 🔐 1. 최소 보장선

* 훈련사 월 지급 하한선 설정 (예: 80만원)
* 미달 시 플랫폼 몫에서 보전

### 🔐 2. 패널티 규칙

* 신고/문제 발생 시:

  * `violation_count` 증가
  * 자동 수익화 OFF
  * 점수 감산

### 🔐 3. 콘텐츠 승인제

* 영상 → 관리자 승인 → 수익화
* 초기에는 **완전 필수**

# 7️⃣ 왜 이 구조가 좋은가

✔ 계약 분쟁 거의 없음
✔ “유튜브처럼” 누구나 이해

✔ 플랫폼은 초기 리스크 회수 가능
✔ Replit → 실제 서비스로 그대로 확장 가능
