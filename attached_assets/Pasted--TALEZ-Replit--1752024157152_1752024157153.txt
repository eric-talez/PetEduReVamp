테일즈(TALEZ) 서비스의 확장 전략으로 제안된 **훈련사 양성 및 인증 시스템**에 대해, 관리자 기반 Replit 구현을 전제로 한 **기획 및 기능 정의**를 아래와 같이 상세히 정리해드립니다.

---

## 📌 1. 시스템 개요

### 목적

* 공식 테일즈 인증 훈련사를 **기관 소속 여부와 무관하게 모집·양성·인증**하고,
* 훈련사 인증 상태를 관리하며,
* **인증 훈련사만 수업 및 콘텐츠 제작에 참여**할 수 있도록 등급화.

### 대상

* **기관 소속 훈련사**: 제휴기관에 속한 훈련사 (내부 관리)
* **비소속 일반 지원자**: 개인 자격으로 인증 신청 → 평가 후 통과 시 인증

---

## 🧭 2. 주요 기능 요약

| 기능구분   | 기능명       | 설명                              |
| ------ | --------- | ------------------------------- |
| 사용자 기능 | 훈련사 신청    | 이름, 연락처, 소속 여부, 기본 정보 입력 후 지원   |
| 관리자 기능 | 신청 목록 조회  | 전체 지원자 리스트 확인 (관리자 인증 필요)       |
| 관리자 기능 | 지원자 상세 평가 | 신청자 세부 내용 열람, 평가 기록 추가          |
| 관리자 기능 | 인증 승인     | 인증 버튼 클릭 시 공식 인증 처리 및 데이터 이동    |
| 공개 기능  | 인증 훈련사 목록 | 현재 인증된 훈련사 리스트 반환 (정렬 및 필터링 가능) |

## 🧩 3. 주요 데이터 모델 정의

### 📌 `TrainerApplication` (지원자 테이블)

| 필드명               | 유형   | 설명                          |
| ----------------- | ---- | --------------------------- |
| `id`              | UUID | 고유 신청 ID                    |
| `name`            | 문자열  | 훈련사 이름                      |
| `email`           | 문자열  | 이메일                         |
| `phone`           | 문자열  | 연락처                         |
| `has_affiliation` | 불린   | 기관 소속 여부                    |
| `status`          | 문자열  | `"pending"` / `"certified"` |
| `submitted_at`    | 날짜   | 신청 일시                       |

---

### 📌 `Certification` (인증 기록 테이블)

| 필드명                | 유형   | 설명                 |
| ------------------ | ---- | ------------------ |
| `certification_id` | UUID | 인증 ID              |
| `trainer_id`       | UUID | 신청자 ID와 연결         |
| `certified_at`     | 날짜   | 인증 일시              |
| `issued_by`        | 문자열  | 인증 발급자 (`admin` 등) |

---

## 🔐 4. 관리자 기능 세부 플로우

### 🧾 신청 → 인증 승인까지 흐름:

1. **지원자 접수**: `/apply_trainer` 엔드포인트로 JSON 형태로 제출
2. **관리자 로그인**: `?admin_key=secret_admin_key` 쿼리 파라미터 사용
3. **신청자 목록 확인**: `/admin/list_applications`
4. **특정 신청자 승인**: `/admin/certify_trainer` POST로 ID 지정
5. **공식 인증자 등록**: `/certified_trainers`에서 공개 확인 가능

## 🛠 5. 구현 기술 기준

| 요소     | 내용                                     |
| ------ | -------------------------------------- |
| 백엔드    | Flask (Replit에서 실행 가능한 경량 구조)          |
| 인증     | 간단한 `admin_key` 인증 → 추후 로그인 세션화 가능     |
| 데이터 저장 | 메모리 기반 (향후 SQLite/Firestore 등으로 확장 가능) |
| 확장성    | 향후 관리자 UI 대시보드, 교육 이수 인증 연동 가능         |
