# TALEZ 권한 시스템 분석 및 기능 연동 상태

## 권한 구조 (상위 → 하위)

### 1. 관리자 (admin)
- **최고 권한**: 모든 기능 접근 가능
- **고유 기능**: 시스템 관리, 기관 관리, 사용자 관리
- **접근 가능한 모든 페이지**: 훈련사, 기관 관리자, 견주 기능 포함

### 2. 기관 관리자 (institute-admin)
- **중간 권한**: 기관 운영 및 소속 훈련사 관리
- **고유 기능**: 기관 시설 관리, 훈련사 관리, 알림장 모니터링
- **제한 사항**: 시스템 전체 관리 불가

### 3. 훈련사 (trainer)
- **전문가 권한**: 교육 서비스 제공 및 학생 관리
- **고유 기능**: 강좌 관리, 학생 관리, 수익 관리
- **제한 사항**: 시스템 관리 및 타 훈련사 정보 접근 불가

### 4. 견주 (pet-owner)
- **일반 사용자**: 교육 서비스 수강 및 반려견 관리
- **고유 기능**: 반려견 관리, 학습 현황, 건강 관리
- **제한 사항**: 관리 기능 접근 불가

### 5. 일반 회원 (비로그인)
- **최소 권한**: 공개 정보 및 서비스 소개만 접근
- **접근 가능**: 홈, 강의 찾기, 전문가 찾기, 커뮤니티 (읽기)
- **제한 사항**: 개인화 기능 및 관리 기능 접근 불가

## 접근 제어 메커니즘

### 1. 사이드바 레벨 제어
- `showAdminMenu`: 관리자 메뉴 표시 여부
- `showTrainerMenu`: 훈련사 메뉴 표시 여부
- `showInstituteMenu`: 기관 관리자 메뉴 표시 여부
- `showPetOwnerMenu`: 견주 메뉴 표시 여부

### 2. 라우팅 레벨 제어
- 로그인 필요 페이지 접근 시 자동 로그인 페이지 리다이렉트
- 권한 부족 시 접근 거부 팝업 표시
- 역할별 접근 제한 체크

### 3. 컴포넌트 레벨 제어
- 각 페이지 컴포넌트에서 권한 체크
- API 호출 시 서버 사이드 권한 검증
- 동적 메뉴 표시/숨김 처리

## 기능 연동 상태 평가

### ✅ 정상 작동 확인된 기능
1. **권한별 메뉴 표시**: 사용자 역할에 따른 메뉴 동적 표시
2. **접근 제어**: 비인증 사용자 로그인 페이지 리다이렉트
3. **권한 제한**: 권한 부족 시 접근 거부 팝업
4. **메뉴 그룹 토글**: 메뉴 그룹 확장/축소 기능
5. **사이드바 확장/축소**: 반응형 사이드바 토글

### ⚠️ 개선 필요 사항
1. **API 인증**: 일부 API 엔드포인트에서 서버 사이드 권한 검증 필요
2. **에러 핸들링**: 권한 오류 시 더 구체적인 메시지 제공
3. **캐싱**: 권한 정보 캐싱으로 성능 최적화
4. **감사 로그**: 권한 변경 및 접근 시도 로그 기록

### 🔧 현재 구현된 보안 기능
- 클라이언트 사이드 권한 체크
- 동적 메뉴 렌더링
- 접근 제한 팝업
- 로그인 상태 검증
- 역할 기반 리다이렉션
