
# 권한별 서비스 연동 관계 및 서비스 플로우

## 1. 권한 체계 개요

### 권한 레벨 (roleHierarchy)
- **admin (100)**: 최고 관리자 - 모든 기능 접근 가능
- **institute-admin (80)**: 기관 관리자 - 소속 기관 내 모든 권한
- **trainer (60)**: 훈련사 - 강의 및 알림장 관리
- **pet-owner (40)**: 반려인 - 일반 사용자 기능
- **user (20)**: 미등록 사용자 - 최소 권한

## 2. 서비스별 권한 매트릭스

### 2.1 알림장 서비스 (`/notebook`)
- **pet-owner**: 자신의 반려동물 알림장 작성/조회
- **trainer**: 담당 반려동물 알림장 작성/조회/수정
- **institute-admin**: 소속 훈련사가 작성한 알림장 조회 가능
- **admin**: 모든 알림장 조회/관리

### 2.2 상담신청 서비스 (`/consultation`)
- **pet-owner**: 상담 신청 가능
- **trainer**: 상담 요청 확인 및 응답
- **institute-admin**: 소속 훈련사 상담 현황 모니터링
- **admin**: 전체 상담 현황 관리

### 2.3 강좌 관리 서비스 (`/courses`)
- **pet-owner**: 강좌 조회 및 수강신청
- **trainer**: 강좌 생성/수정/관리
- **institute-admin**: 소속 훈련사 강좌 승인/관리
- **admin**: 모든 강좌 관리

### 2.4 메시징 서비스 (`/messaging`)
- **pet-owner**: 훈련사와 1:1 메시지
- **trainer**: 수강생과 메시지 교환
- **institute-admin**: 소속 구성원 메시지 모니터링
- **admin**: 전체 메시지 관리

## 3. 권한 검증 플로우

### 3.1 인증 플로우
```
1. 로그인 요청 → 2. 인증 서버 검증 → 3. JWT 토큰 발급 → 4. 세션 저장
```

### 3.2 권한 검증 플로우
```
1. API 요청 → 2. 토큰 검증 → 3. 권한 레벨 확인 → 4. 서비스 접근 허용/거부
```

## 4. 서비스 연동 관계도

### 4.1 알림장 서비스 연동
- **Auth Service**: 권한 검증
- **Notification Service**: 알림 발송
- **AI Service**: AI 알림장 생성
- **File Upload Service**: 미디어 첨부

### 4.2 상담 서비스 연동
- **Auth Service**: 권한 검증
- **Messaging Service**: 상담 메시지
- **Notification Service**: 상담 알림
- **Video Call Service**: 화상 상담

## 5. 개선 사항

### 5.1 권한 관리 통합
- 모든 서비스에서 일관된 권한 체크
- 권한별 메뉴 표시/숨김 처리
- API 레벨 권한 검증 강화

### 5.2 서비스 플로우 최적화
- 권한에 따른 적절한 리다이렉션
- 에러 처리 및 사용자 피드백 개선
- 로딩 상태 및 접근성 향상
